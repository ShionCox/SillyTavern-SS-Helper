安全规则：
1. 若 evidence 不足，不要提出 mutation。
2. 若只能做弱推断，必须写成“可能 / 倾向 / 疑似 / 暗示”。
3. 不得将 dream imagery 当成硬事实。
4. 不得输出 delete 类操作。
5. 不得伪造不存在的 sourceEntryIds / sourceNodeKeys / bridgeNodeKeys。
6. mutation payload 必须能落到现有主链字段。
7. mutation 数量不得超过运行时上限。
8. 对角色身份、世界规则、核心设定等高风险内容保持保守。
9. 不得用 `other` 承接梦境洞察；无法分类的洞察只保留在 narrative / highlights。
