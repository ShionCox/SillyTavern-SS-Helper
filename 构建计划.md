## P0：先补到“稳定可用 Beta”

### 1. 聊天删除闭环

目标：用户在 SillyTavern 删除聊天后，不留下孤儿记忆。

要做的事：

* 在 `MemoryOS/src/index.ts` 增加“聊天删除/聊天列表变化”监听。
* 新增统一清理入口，例如：

  * `MemoryOS/src/core/chat-lifecycle-manager.ts`
  * 或在 `MemoryOS/src/sdk/memory-sdk.ts` 增加 `purgeChat()` / `archiveChat()`
* 清理范围至少包括：

  * `events`
  * `facts`
  * `world_state`
  * `summaries`
  * `templates`
  * `meta`
  * `template_bindings`
  * `vector_*`
  * `chatState`
  * SDK 的 `chat_documents / chat_plugin_state / chat_plugin_records`
* 支持两种模式：

  * `soft_delete`
  * `immediate_purge`
* 删除后同步：

  * 清理 `STX.memory`
  * 清理当前 chat 缓存
  * 清理 LLMHub chat snapshot

原因：删除策略类型已经存在，但宿主删除后的真正闭环还不完整 

---

### 2. 把“多少楼总结一次”正式做成可配置

目标：把已有底层能力变成用户真能调的功能。

要做的事：

* 继续使用现有 `SummaryPolicyOverride`

  * `interval`
  * `windowSize`
  * `enabled` 
* 在 `MemoryOS/src/ui/chatStrategyPanel.ts` 增加输入项：

  * `summaryEnabled`
  * `summaryFloorInterval`
  * `summaryWindowSize`
* 保存时调用：

  * `chatState.setSummaryPolicyOverride(...)`
* 在 `ExtractManager` 继续沿用现有读取逻辑，不需要重写触发链 

推荐默认值：

* 总结开关：开
* 每 12 楼总结一次
* 总结窗口：最近 40 楼

原因：楼层总结链已经落地，只差 UI 出口 

---

### 3. 增加“多少楼刷新一次画像/质量”

目标：让画像和质量分不是每次都重算，也不是永远不更新。

新增设置：

* `profileRefreshInterval`
* `qualityRefreshInterval`

建议行为：

* 每 `N` 个 assistant turn：

  * 刷新 `AdaptiveMetrics`
  * 重算 `ChatProfile`
* 每 `M` 个 assistant turn：

  * 重算 `MemoryQuality`
  * 更新 `MaintenanceAdvice`

推荐默认值：

* 画像刷新：6 楼
* 质量刷新：12 楼

改动点：

* `MemoryOS/src/types/chat-state.ts`
* `MemoryOS/src/core/chat-state-manager.ts`
* `MemoryOS/src/core/extract-manager.ts`
* `MemoryOS/src/ui/chatStrategyPanel.ts`

---

### 4. 修正 LLMHub 的 chat snapshot 写入

目标：避免状态语义混乱。

当前风险点：
`LLMHub/src/index.ts` 的 `persistChatSnapshot()` 把 `state` 和 `summary` 一起塞进第三个参数了 

建议改成：

```ts
writeSdkPluginChatState(
  'stx_llmhub',
  chatKey,
  {
    routeSnapshot: {...},
    budgetSnapshot: {...},
    profile: ...,
  },
  {
    summary: {
      resource: ...,
      model: ...,
    },
  }
);
```

## P1：补齐 SillyTavern 语义适配

### 5. 支持 swipe / regenerate / edit 的记忆语义

目标：数据库记录更接近用户眼里的“当前聊天内容”。

先做轻量版：

* 给事件增加可选字段：

  * `messageLifecycle`
  * `supersededBy`
  * `editedFrom`
* 被 swipe 掉的旧助手回复：

  * 不立刻删原始 event
  * 但在摘要/长期事实提取时默认降权或忽略
* 编辑后的消息：

  * 标记旧版本失效
  * 新版本允许重抽取

改动点：

* `MemoryOS/src/index.ts`
* `MemoryOS/src/core/events-manager.ts`
* `MemoryOS/src/core/extract-manager.ts`
* `MemoryOS/src/core/chat-strategy-engine.ts`

### 6. 新聊天冷启动

目标：新 chat 一开始就更像 ST 角色聊天，而不是空壳库。

第一阶段先做非 AI 版：

* 创建 chat 时抓取：

  * 角色卡基础信息
  * group members
  * lorebook refs
  * first message
  * system prompt 引用
* 存为 `seed bundle`
* 建立弱实体锚点：

  * 当前角色
  * 用户
  * 群成员
  * 世界名

改动点：

* `MemoryOS/src/index.ts`
* `MemoryOS/src/sdk/memory-sdk.ts`
* 新增：

  * `MemoryOS/src/core/chat-bootstrap-manager.ts`

第二阶段再做 AI bootstrap：

* 角色稳定 facts
* 初始 world state
* 初始 profile refine

### 7. 群聊分轨

目标：群聊不要把多人状态混成一锅。

改动点：

## P2：提升产品体验

### 8. 让设置更像 ST 用户语言

目标：减少“技术参数感”。

在聊天策略面板新增用户向导项：

* 轻陪伴
* 长剧情 RP
* 世界设定问答
* 群聊/跑团
* 工具问答

选择预设后自动映射到：

* `chatType`
* `stylePreference`
* `memoryStrength`
* `extractStrategy`
* `summaryStrategy`
* `vectorStrategy`

改动点：

* `MemoryOS/src/ui/chatStrategyPanel.ts`

---

### 9. 角色级默认策略

目标：同一个角色的新聊天自动继承风格。

新增三级策略：

* 全局默认
* 角色默认
* 聊天覆盖

建议：

* 角色默认先存到 SDK chat shared signal 或单独 settings namespace
* 新 chat 创建时应用到 `chatState`

---

### 10. 聊天生命周期状态

目标：针对不同阶段自动调整策略。

新增状态：

* `new`
* `active`
* `stable`
* `long_running`
* `archived`

自动行为：

* `new`：轻量摘要，不开向量
* `stable`：开启质量监控
* `long_running`：定期压缩/重建摘要
* `archived`：不注入，只保留
