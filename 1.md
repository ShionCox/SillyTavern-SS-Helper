

# 我给你的最终方案

## 总原则

做三层防护，缺一不可：

### 第一层：生成后立刻清理原消息文本

把 assistant 输出里的 `rolljson` 真正从文本里删掉，不是隐藏 DOM，不是只在前端看不到。

### 第二层：所有可读字段同步写回

不只改 `mes`，还要改：

* `mes`
* `content`
* `text`
* `swipes[swipe_id]`
* 如果 swipe 是对象，还要改它的 `mes/content/text`

这一步是最关键的，因为很多总结插件不会读你现在屏幕上显示的内容，而是直接读消息对象。

### 第三层：对内部消息/事件加跳过摘要标记

给 RollHelper 自己生成的内部说明、摘要块、事件块统一加：

* `_skipHorae = true`
* `extra.skip_summary = true`
* `extra.skip_memory = true`

Horae 如果识别这些标记，就直接跳过；就算它某些链路不认，你前两层也已经把文本清干净了。

---

# 为什么这是“最好”的方案

因为它同时解决了你现在最可能遇到的 3 类问题：

## 1. UI 删了，但数据还在

你看到没了，不代表 Horae 读不到。

## 2. `mes` 删了，但 `swipes` 还在

这是 SillyTavern 插件兼容里非常常见的问题。
很多插件显示时读当前 swipe，保存时改主字段，结果两边不一致。

## 3. RollHelper 清理时机晚于 Horae 摘要

如果 Horae 在“生成完成事件”上抢先读取到了原始回复，哪怕你后面删掉了，Horae 也已经总结进去了。

所以最好的方案不是只改一个函数，而是让 `RollHelper` 自己具备“**任何摘要插件都尽量读不到 rolljson**”的能力。

## 一、做一个统一的文本读写层

你现在不要在各个事件里随手改 `message.mes = ...` 了。
统一收口成两个函数：

* `getMessageTextSafe`
* `setMessageTextSafe`

---

### 1）统一读取当前真实文本

```ts
export function getMessageTextSafe(message: any): string {
  if (!message) return '';

  // 优先读取当前 swipe
  if (Array.isArray(message.swipes) && typeof message.swipe_id === 'number') {
    const id = message.swipe_id;
    if (id >= 0 && id < message.swipes.length) {
      const swipe = message.swipes[id];

      if (typeof swipe === 'string') return swipe;
      if (swipe && typeof swipe === 'object') {
        if (typeof swipe.mes === 'string') return swipe.mes;
        if (typeof swipe.content === 'string') return swipe.content;
        if (typeof swipe.text === 'string') return swipe.text;
      }
    }
  }

  if (typeof message.mes === 'string') return message.mes;
  if (typeof message.content === 'string') return message.content;
  if (typeof message.text === 'string') return message.text;

  return '';
}
```

---

### 2）统一写回所有字段

```ts
export function setMessageTextSafe(message: any, newText: string) {
  if (!message) return message;

  if (typeof message.mes !== 'undefined') {
    message.mes = newText;
  }

  if (typeof message.content !== 'undefined') {
    message.content = newText;
  }

  if (typeof message.text !== 'undefined') {
    message.text = newText;
  }

  if (Array.isArray(message.swipes) && typeof message.swipe_id === 'number') {
    const id = message.swipe_id;
    if (id >= 0 && id < message.swipes.length) {
      const swipe = message.swipes[id];

      if (typeof swipe === 'string') {
        message.swipes[id] = newText;
      } else if (swipe && typeof swipe === 'object') {
        if (typeof swipe.mes !== 'undefined') swipe.mes = newText;
        if (typeof swipe.content !== 'undefined') swipe.content = newText;
        if (typeof swipe.text !== 'undefined') swipe.text = newText;
      }
    }
  }

  return message;
}
```

## 二、做一个专门的 rolljson 净化器

这个函数不要只识别最理想情况。
最好的做法是：

### 强删 3 类块

1. `rolljson ... `
2. 你自己管理的 summary/managed block
3. 明显是 dice 事件的 json 代码块

---

### 推荐实现

````ts
export function stripRollJsonBlocks(text: string): string {
  if (!text) return text;

  let output = text;

  // 1. 强删明确的 rolljson 块
  output = output.replace(/```rolljson\s*[\s\S]*?```/gi, '');

  // 2. 删除带 dice_events 的 json 块
  output = output.replace(/```json\s*([\s\S]*?)```/gi, (full, body) => {
    if (/"type"\s*:\s*"dice_events"/i.test(body)) {
      return '';
    }
    return full;
  });

  // 3. 删除 html/pre 包裹的事件块
  output = output.replace(/<pre[^>]*>[\s\S]*?"type"\s*:\s*"dice_events"[\s\S]*?<\/pre>/gi, '');

  // 4. 删除你自己注入的 managed summary block（按你的标记名改）
  output = output.replace(
    /<!--\s*ROLLHELPER_SUMMARY_START\s*-->[\s\S]*?<!--\s*ROLLHELPER_SUMMARY_END\s*-->/gi,
    ''
  );

  // 5. 清理多余空行
  output = output.replace(/\n{3,}/g, '\n\n').trim();

  return output;
}
````

## 三、在“生成结束”事件里做前置净化

这是整个兼容的核心落点。
你仓库里本来就有 `hooks / parser / render / summary` 这类事件分层，所以最适合把这个兼容逻辑挂在 **generation ended** 的消息落库前。

你应该把现在“解析事件 + UI隐藏”的逻辑，升级成下面这个顺序：

```ts
function sanitizeAssistantMessageForSummary(message: any) {
  if (!message) return message;

  const rawText = getMessageTextSafe(message);
  const cleanedText = stripRollJsonBlocks(rawText);

  setMessageTextSafe(message, cleanedText);

  // 双保险：加跳过摘要标记
  message._skipHorae = true;
  message.extra = {
    ...(message.extra || {}),
    rollhelper_sanitized: true,
    skip_summary: true,
    skip_memory: true,
  };

  return message;
}
```

然后在你的生成结束处理里：

```ts
function handleGenerationEnded(...) {
  const message = getLatestAssistantMessageSomehow();
  if (!message) return;

  // 先解析 roll 数据，保留你现有逻辑
  const rawText = getMessageTextSafe(message);
  const parsed = parseEventEnvelopesEvent(rawText);

  // 这里继续做你的骰子事件入库、状态更新等
  applyDiceEvents(parsed);

  // 然后立刻净化文本，避免被摘要插件看到
  sanitizeAssistantMessageForSummary(message);

  // 再做持久化
  persistChatSafeEvent?.();

  // 最后才是 UI 层隐藏
  hideEventCodeBlocksInDomEvent?.();
}
```

# 四、把“显示隐藏”和“数据删除”彻底分离

你要明确区分两件事：

## 以前

* 解析到了 `rolljson`
* 在 DOM 里隐藏掉
* 也许顺便改一下 `mes`

## 现在

* 解析到了 `rolljson`
* 先做事件处理
* **从消息数据中彻底删除**
* 同步写回所有字段
* 持久化
* 最后再做 UI 隐藏

也就是说，**DOM 隐藏只能算美化，不能算兼容手段**。

# 五、内部总结块不要混进普通聊天文本

你仓库里本身还有 `summaryEvent` 这类东西，说明 RollHelper 自己也会构造一些摘要性文本块。
这类内容如果最终也被 Horae 再总结一遍，就会出现“双重总结污染”。

所以最好的改法是：

## 对 RollHelper 自己插入的摘要块

统一走单独字段或 metadata，不直接混到聊天正文里。
实在必须混正文，就至少包一层你自己的受控标记，然后在摘要前剥掉。

例如：

```ts
export function wrapInternalBlock(content: string) {
  return [
    '<!-- ROLLHELPER_INTERNAL_START -->',
    content,
    '<!-- ROLLHELPER_INTERNAL_END -->',
  ].join('\n');
}
```

然后净化函数里加：

```ts
output = output.replace(
  /<!--\s*ROLLHELPER_INTERNAL_START\s*-->[\s\S]*?<!--\s*ROLLHELPER_INTERNAL_END\s*-->/gi,
  ''
);
```

# 六、不要只依赖 `"type":"dice_events"` 才删除

这个点也要升级。

因为一旦 Horae 或别的系统提示词影响了模型输出格式，比如：

* 改了字段名
* 多包了一层对象
* 输出成 `rolljson` 但没写 `type: "dice_events"`

你现在的删除规则就会漏掉。

所以最好的删除优先级应该是：

## 删除优先级

### 第一优先：语言标签是 `rolljson`

直接删，不犹豫。

### 第二优先：json 里包含明显骰子结构

比如 `dice_events`、`rolls`、`check_result` 等。

### 第三优先：你自己生成过的内部标记块

直接删。

也就是：

```ts
export function isDiceLikeJson(body: string): boolean {
  return [
    /"type"\s*:\s*"dice_events"/i,
    /"rolls"\s*:/i,
    /"check_result"\s*:/i,
    /"dice"\s*:/i,
    /"dc"\s*:/i,
  ].some((re) => re.test(body));
}
```


# 我给你的最终结论

## 最好的兼容修改方案就是：

### 以 RollHelper 为主，做“摘要插件无感净化层”

包括这 4 件事：

1. **生成结束后先解析，再立刻净化原消息**
2. **统一同步 `mes/content/text/swipes`**
3. **对内部消息加 `_skipHorae / skip_summary / skip_memory`**
4. **把 DOM 隐藏从“主逻辑”降级为“显示优化”**

这样改完后，兼容的就不只是 Horae，基本所有读聊天历史做总结/记忆的插件都会更稳。
