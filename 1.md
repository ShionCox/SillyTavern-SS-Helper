# RollHelper 数据库彻底换模实施蓝图

## Summary
这次不是“改几个字段”，而是 **整包换模**。从当前代码看，`pendingRound / summaryHistory / blindHistory / message extra / chat_plugin_records` 已经深度渗透到 `storeEvent / hooksEvent / roundEvent / promptEvent / anchorEvent / interactiveTriggersEvent / commands / runtime` 里，所以要真正做到：

- `chatId` 作为唯一聊天主键
- `messageId` 作为唯一 AI 楼层 `floorId`
- `chatData[chatId]` 成为唯一真源
- 旧链路、旧字段、旧兼容、旧恢复逻辑全部删除

实施上必须按“**先换数据模型，再换生命周期，再换 UI 恢复，再删旧接口**”四段走，不然会出现一半代码读新结构、一半代码还写旧结构的撕裂状态。

## 现状边界
当前旧结构的核心入口集中在这些文件：

- `RollHelper/src/settings/storeEvent.ts`
  这里现在的聊天级持久化主结构还是：
  - `pendingRound`
  - `summaryHistory`
  - `blindHistory`
  - `activeStatuses`
  - `lastBaseRoll`
- `RollHelper/src/types/eventDomainEvent.ts`
  这里的 `DiceMetaEvent` 和大量领域类型仍围绕 `pendingRound / summaryHistory / blindHistory`
- `RollHelper/src/events/roundEvent.ts`
  当前公开骰、暗骰、轮次汇总、历史归档都以旧模型组织
- `RollHelper/src/events/promptEvent.ts`
  当前 prompt 注入仍直接消费 `pendingRound / summaryHistory / blindHistory`
- `RollHelper/src/events/anchorEvent.ts`
  当前历史卡片恢复完全依赖 `pendingRound + summaryHistory`
- `RollHelper/src/events/interactiveTriggersEvent.ts`
  当前 trigger 已结算态恢复会扫 `pendingRound + summaryHistory + blindHistory`
- `RollHelper/src/events/messageSanitizerEvent.ts`
  当前原始源文本快照还落在消息 `extra`
- `RollHelper/src/events/interactiveTriggerMetadataEvent.ts`
  当前 trigger metadata 还落在消息 `extra`
- `RollHelper/src/runtime/eventRuntime.hooks.ts`
  当前初始化恢复还是 `loadChatScopedStateIntoRuntimeEvent -> refreshAllWidgetsFromStateEvent`
- `RollHelper/src/commands/*.ts`
  命令层也直接读旧 `meta`

这意味着要做到你要求的“彻底替换、不兼容”，必须把以上这些点一起切掉。

```json
        "12": {

```

固定规则：

- `chatData` 一级 key 直接用官方 `chatId`
- `floors` 一级 key 直接用 AI `messageId`
- 只存 assistant 楼层
- user 楼层不入 `floors`，只更新 `meta.lastUserMessageId`
- `raw + processed` 保留
- `statuses.current.snapshot` 保留
- 其他旧缓存全部不保留

## 生命周期重绑
新结构对应的读写责任必须固定成下面这套：

### 1. `CHAT_CHANGED`
- 收到事件后，立即 `SillyTavern.getContext().chatId`
- 用这个 `chatId` 载入 `chatData[chatId]`
- 初始化运行时只保留当前聊天对象，不再反序列化 `pendingRound / summaryHistory / blindHistory`

### 2. `MESSAGE_SENT(messageId)`
- 不创建楼层存储
- 只更新：
  - `chatData[chatId].meta.lastUserMessageId = messageId`

### 3. `MESSAGE_RECEIVED(messageId)`
- `messageId` 直接视为 `floorId`
- 只清理当前 `floors[floorId]` 旧数据：
  - `content`
  - `triggers`
  - `eventDice`
  - `statusRefs`
  - `roundRefs`
- 同时从：
  - `rounds.records[*].floorIds / eventRefs / rollRefs`
  - `statuses.records`
  中移除该楼层旧引用
- 不做 sanitize
- 不做 UI
- 不做 attach
- 不做 auto roll

### 4. `GENERATION_ENDED`
- 以当前 `floorId` 作为唯一提交点
- 完整执行：
  - 读取原文
  - 生成 `processed`
  - 解析 trigger
  - 解析事件
  - 写入 `floors[floorId]`
  - 执行 auto roll
  - 更新 `rounds.records`
  - 更新 `statuses.records/current`
  - 最后统一渲染：
    - 结果卡
    - 暗骰卡
    - `rh-trigger`
    - 原格式按钮
    - 代码块隐藏

## 必删旧结构
这次必须整体删除，不保留兼容入口：

### 持久化层删除
- `pendingRound`
- `summaryHistory`
- `blindHistory`
- `lastProcessedAssistantMsgId` 这类基于旧 round/history 恢复链的持久化派生值
- `chat_plugin_records` 里为 `blind_results / round_summaries` 服务的正式恢复链
- 消息 `extra` 中作为正式真源的：
  - `rollhelper_interactive_triggers_v1`
  - `rollhelper_trigger_pack_v1`
  - `rollhelper_interactive_triggers_source_state_v1`
  - `rollhelper_original_source_v1`
  - `rollhelper_original_source_enabled_v1`
  - `rollhelper_original_source_meta_v1`

### 主键/标识删除
- `resolveCurrentChatKeyEvent()` 当前那套 `buildTavernChatScopedKeyEvent(...)`
- 自定义 `sourceFloorKey`
- 依赖 `assistantMsgId` 作为持久化主键的逻辑

### 恢复链删除
- `loadChatScopedStateIntoRuntimeEvent()` 当前 pending/history/blindHistory 装载模型
- `refreshAllWidgetsFromStateEvent()` 基于 pending/history 的恢复逻辑
- `anchorEvent.ts` 中基于 `pendingRound + summaryHistory` 的挂载策略
- `interactiveTriggersEvent.ts` 中扫旧 `meta` 结构恢复 blind/trigger 的逻辑

真正落地时，建议按下面顺序改：

### 第一组：定义新模型

目标：

目标：
- 所有生命周期都改为直接操作 `chatData[chatId]`
- 删除旧 `meta.pendingRound / summaryHistory / blindHistory` 读写
- 聊天进入时只加载当前 `chatData[chatId]`
- UI 恢复改为“从 `floors + rounds` 直接挂载”

### 第三组：改业务层
- `RollHelper/src/events/roundEvent.ts`
- `RollHelper/src/events/promptEvent.ts`

- 不再从旧 `meta` 中拼接 UI

## 建议的实际实施顺序
如果下一步真的开始改代码，最稳的顺序是：

1. 先把 `types + storeEvent` 全部换成新模型
2. 再把 `hooksEvent` 的三段式生命周期改成只读写新模型
3. 再改 `roundEvent / promptEvent / interactiveTriggersEvent / messageSanitizerEvent`
4. 最后改 `anchorEvent / renderEvent / commands`
5. 最后一次性删除旧字段、旧 helper、旧测试

不能反过来，因为 UI 和命令层现在都依赖旧真源。

## Assumptions
- 旧聊天数据全部放弃
- 不保留迁移
- 不保留双写

## 结论
这次重构是 **全模型替换**，不是在 `storeEvent.ts` 里改一个 JSON 就够了。  
如果按你现在的要求真落地，最少要同步重写：

- `storeEvent.ts`
- `eventDomainEvent.ts`
- `hooksEvent.ts`
- `eventRuntime.hooks.ts`
- `roundEvent.ts`
- `promptEvent.ts`
- `interactiveTriggersEvent.ts`
- `messageSanitizerEvent.ts`
- `anchorEvent.ts`
- `renderEvent.ts`
- 相关 commands 与 tests

这套蓝图已经和当前代码的真实耦合点对上了，后面一旦切回执行模式，就应该按这份清单直接开始拆模块，而不是再走过渡层。
