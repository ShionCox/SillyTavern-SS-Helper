重构一下，流传输的消息不需要了，都移动到GENERATION_ENDED 里。就是在GENERATION_ENDED 。我这里重新整理一下链路：
GENERATION_STARTED：
type 作为第一个参数发出去的，
type === 'normal'：普通生成，通常就是新楼层
type === 'regenerate'：点了 Regenerate，属于当前楼层重生成
type === 'swipe'：点了 Swipe，也是当前楼层换一个版本
type === 'continue'：对当前最后一条继续写，不是新楼层重开
type === 'quiet'：后台生成，通常不是用户手动发言触发的普通楼层生成。
如果type是normal的话，就按新楼层逻辑来。
如果type是swipe的话，就先清理掉当前楼层所有卡片。
MESSAGE_RECEIVED：
第一个参数就是 this.messageId，也就是那条消息在当前聊天里的索引
获取到楼层之后，就可以删除当前楼层的骰子数据，暗骰数据。不是删除全部，是从数据库里删除当前楼层对应的数据。
GENERATION_ENDED
处理完所有的UI相关的，比如说rolljson和triggerjson的移除，卡片的渲染，rh-trigger 的渲染。
