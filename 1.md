# RollHelper 沉浸式判定闭环方案

## Summary
目标是把 `RollHelper` 从“显式按钮掷骰器”升级成“叙事驱动判定层”：可交互检定入口由 AI 在回复中显式标记，玩家通过点击高亮词或划词触发检定；默认敏感技能走暗骰；被动检定继续从世界书隐藏信息中静默解锁；极端结果和失败结果通过更强的 Prompt 约束强制转化为叙事后果。

本版按以下决策实现：
- 高亮来源：世界书/作者控制为主
- 高亮判定：AI 显式打标
- 交互方式：同时支持点击高亮词和划词
- 范围：一次打通“高亮触发 + 默认暗骰技能 + 被动世界书 + 后果约束”的完整闭环

## Key Changes
### 1. 交互入口从“按钮”改为“叙事片段”
- 在现有 prompt 协议中新增一段轻量交互标记规范，要求 AI 在回复正文中只对“允许玩家发起检定的词或短语”加标记，而不是整句乱标。
- 标记内容至少包含：
  - `label`：玩家看到的高亮词
  - `action`：建议动作，如 `调查 / 回忆 / 洞察 / 搜查`
  - `skill`：对应技能
  - `blind`：是否默认暗骰
  - `sourceId`：稳定标识，便于去重和后续上下文引用
  - 可选 `dcHint / loreType / note`
- 前端在消息渲染后把这些标记转成高亮 span，不直接暴露协议痕迹；高亮样式用微弱下划线或柔和发光，默认不做强侵入色块。
- 点击高亮词时，在鼠标附近弹出轻量 Tooltip / Context Menu，菜单项直接是“动作 + 技能 + 是否暗骰”。
- 划词触发只对短文本生效：
  - 若划中的文本本身已包含高亮节点，则优先复用该节点的推荐动作
  - 若是普通文本，则走保守兜底，只提供少量通用动作，不允许无限刷技能列表
- 首版不保留原来那种面向空气的一排常驻技能按钮；保留 `/roll`、`/broll` 作为高级/手动兜底入口。

### 2. 暗骰升级为“默认技能策略 + 触发态”
- 设置项新增“默认暗骰技能列表”，首版内置推荐：`洞察 / 潜行 / 搜查 / 历史 / 调查`，可编辑。
- 由高亮菜单触发时，若命中默认暗骰技能，菜单直接显示 `调查（暗骰）` 这类动作；玩家无需额外切换模式。
- 暗骰结果卡继续隐藏真实点数、成功失败、差值与原始骰面，只展示统一文案和神秘视觉态。
- Prompt 注入继续走 `<dice_blind_guidance>`，但文案加强成“DM 叙事指令”而不是普通结果摘要：
  - 成功时要求自然揭示信息
  - 失败时允许给出错误线索、遗漏、误判
  - `nat1/nat20` 必须使用更强烈的叙事指令
- 高亮触发的暗骰需要记录来源片段，便于 AI 知道“玩家是在检查哪个词/哪段线索”。

### 3. 被动检定与世界书联动收敛为同一套叙事入口
- 继续沿用现有 `RH_PASSIVE` 世界书块格式，不改基础结构。
- 被动命中后，不仅把隐藏信息注入 AI，也允许 AI 在下一条回复里把其中可交互的关键词按新协议打标，这样被动发现能自然过渡到主动检定。
- 世界书完整样例里补充“被动发现 + 后续可点击词”的写法示例，明确作者如何把隐藏发现与后续调查入口串起来。
- 被动发现缓存、`once/persistent`、按 `priority/DC` 排序的现有逻辑保持不变；新增要求是“已发现内容在后续回复可继续被 AI 重用为交互片段”。

### 4. 后果约束从“结果提示”升级成“叙事规则”
- 在现有 `<dice_result_guidance>` 和 `<dice_blind_guidance>` 基础上新增统一的“后果约束模板”：
  - 大成功：必须产生额外收益、额外线索或更优局面
  - 普通成功：允许稳定推进，但不得无缘无故追加重罚
  - 普通失败：必须有明确叙事代价，至少是时间损失、误判、暴露、资源消耗、引发下一风险之一
  - 大失败：必须出现显著误判或危险后果，但不能把剧情直接卡死
- 失败代价要和触发动作绑定，而不是模板化胡乱惩罚；例如搜查失败更偏向误触陷阱、惊动守卫、漏掉线索。
- 首版不新增复杂规则编辑器，只在设置页提供一个简洁的“失败必须附带叙事代价”开关和一段说明文案；具体模板仍由 prompt 层生成。

### 5. 实现落点
- Prompt 层：
  - 扩展 `promptEvent.ts` 的系统协议，加入“交互标记输出规范”和更强的后果约束文本。
- 消息后处理层：
  - 新增一个消息增强模块，监听新助手消息、swipe 变化和编辑后重渲染，把 AI 标记转为高亮节点。
- 交互层：
  - 复用 `_Components/sharedTooltip` 或 `_Components/sharedContextMenu` 做悬浮动作菜单。
  - 复用 `_Components/sharedFloatingPanel` 思路，但这里保持 Tooltip 级轻量，不做大窗。
- 数据与设置：
  - 扩展 `eventDomainEvent.ts`、设置 store 和设置 UI，增加默认暗骰技能列表、交互标记开关、失败代价约束开关。
- 结算层：
  - 高亮菜单触发最终仍落到现有 `/roll` / `/broll` 对应的底层结算函数，避免新开第二套骰子系统。

## Public Interfaces / Types
- 新增一类 AI 交互标记数据结构，例如 `InteractiveTriggerEvent`：
  - `triggerId`
  - `label`
  - `action`
  - `skill`
  - `blind`
  - `sourceMessageId`
  - `sourceId`
  - `textRange` 或等价定位字段
- 设置项新增：
  - `enableInteractiveTriggers`
  - `interactiveTriggerMode: "ai_markup"`
  - `defaultBlindSkillsText`
  - `enableNarrativeCostEnforcement`
- Prompt 协议新增一段明确的 AI 输出约束：
  - 只标真正可交互的线索词
  - 标记数量受限，避免全篇发光
  - 被动发现内容可继续标记
  - 标记必须和当前叙事上下文对应

## Test Plan
- 高亮触发
  - AI 回复含合法交互标记时，前端能正确转成高亮词
  - 点击高亮词能弹出菜单并触发正确技能
  - 划中高亮短语时复用同一触发信息
  - swipe 切换、消息重渲染后不会重复嵌套高亮
- 暗骰闭环
  - 默认暗骰技能从高亮菜单触发时走隐藏结果卡
  - AI 下一轮能收到真实结果和来源片段
  - 玩家界面无真实点数泄漏
- 被动联动
  - 世界书 `RH_PASSIVE` 命中后 AI 能收到隐藏信息
  - AI 后续能把被动发现中的关键词标成交互入口
  - 未命中的隐藏信息不显示、不标记
- 后果约束
  - 普通失败会生成带叙事代价的 prompt 指令
  - 大失败/大成功会生成强化约束
  - 暗骰失败时允许误导信息而不是简单“一无所获”
- 兼容性
  - 不破坏现有事件卡、`/roll`、`/broll`、summary、worldbook 写回
  - 移动端 Tooltip 不遮挡主要阅读区域，至少可点击关闭与重新触发

## Assumptions
- 首版高亮由 AI 显式打标，前端不做自由文本智能识别。
- 点击和划词都支持，但点击高亮词是主路径；普通划词只做保守兜底，不开放任意技能刷骰。
- 世界书仍使用现有 `RH_PASSIVE` 注释块格式，不改为新 DSL。
- 默认暗骰技能列表可配置，但首版只支持按技能名匹配，不做更复杂的场景规则。
- 后果约束通过 Prompt 实现，不在本版加入独立规则脚本引擎。
